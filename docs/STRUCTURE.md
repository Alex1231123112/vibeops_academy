# Структура проекта: внутреннее vs публичное

## Что пушим в Git (публичный репозиторий)

Всё, что видят ученики и контрибьюторы:

| Папка/файл | Назначение |
|------------|------------|
| `module-*/` | Уроки курса |
| `resources/translations/` | Переводы материалов |
| `resources/videos.md` | Подборка видео |
| `resources/posts/` | **Шаблоны** постов (без дат, без личных заметок) |
| `prompts-library/` | Библиотека промптов |
| `cheatsheets/` | Шпаргалки |
| `templates/` | Шаблоны проектов |
| `tools/` | Инструменты (админка, HT) |
| `community/` | FAQ, как задавать вопросы |
| `integrations/` | agency-agents и др. |
| `docs/` | Документация (кроме internal) |
| `README.md`, `CONTRIBUTING.md`, `ROLES.md` | Основные файлы |
| `TELEGRAM_CONTENT.md` | Контент-план и шаблоны |

## Что НЕ пушим (только для себя)

Папка `internal/` — в `.gitignore`. Храните там:

| Содержимое | Примеры |
|------------|---------|
| Черновики постов | Посты с датами, перед публикацией |
| Планирование | Контент-календарь с датами |
| Заметки | Идеи, TODO, заметки по урокам |
| Локальные настройки | Ссылки на свой репо, личные контакты |
| Отчёты | Статистика канала, аналитика |

### Структура internal/ (создайте сами)

```
internal/
├── posts/           # Черновики постов перед публикацией
├── planning/        # Контент-календарь, планы
├── notes/           # Личные заметки
└── .gitkeep         # (опционально) чтобы папка создалась
```

## Правило

**Перед коммитом:** всё личное, с датами или конфиденциальное — в `internal/`. В Git — только то, что можно показывать публике.
