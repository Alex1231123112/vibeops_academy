# Чек-лист финального проекта

## ⏱️ Сколько времени

Ориентир: **6–12 часов** на весь проект (зависит от масштаба MVP).

## ✅ Чек-лист (junior-ready)

### Репозиторий и документация

- [ ] Код в GitHub (публичный или по договорённости)
- [ ] README: что это, как запустить, как прогнать тесты, env vars (без секретов)

### Приложение (prod-minimum)

- [ ] Есть `/health` endpoint (200 + JSON)
- [ ] Есть базовые логи (старт + ошибки; без секретов)

### Docker (воспроизводимость)

- [ ] `Dockerfile` собирается и запускает приложение
- [ ] `docker-compose.yml` поднимает приложение (и зависимости, если есть)
- [ ] Dockerfile не запускает приложение от root (есть `USER`)
- [ ] Есть `.dockerignore` (не утекают `.env`, `.git`, `.venv`, `__pycache__` и т.п.)

### CI/CD

- [ ] Есть CI: GitHub Actions запускает тесты на `push` и `pull_request`
- [ ] Есть CD: GitHub Actions деплоит на `main` (или есть чёткая инструкция деплоя)
- [ ] Секреты только в GitHub Secrets / переменных окружения платформы

### Доступность

- [ ] Приложение доступно по HTTPS (есть ссылка)

### Тесты

- [ ] Есть базовые тесты (минимум: 2–3 unit-теста и/или 1 smoke-тест)

## 🧪 Test plan (как проверять)

### Локально (до пуша)

- [ ] `pytest` (или эквивалент) проходит
- [ ] `docker build ...` проходит
- [ ] `docker compose up` поднимает сервис, `/health` отвечает

### В GitHub Actions (после пуша)

- [ ] CI workflow зелёный
- [ ] В логах нет ошибок установки зависимостей / запуска тестов

### После деплоя

- [ ] HTTPS URL открывается
- [ ] `/health` возвращает ожидаемый JSON
- [ ] В логах видно “service_started” и нет циклических падений

## Артефакты сдачи

- Ссылка на GitHub репозиторий
- Ссылка на HTTPS сервис
- Скрин/лог CI (или ссылка на Actions)
