# Структура проекта (шаблон)

Это рекомендуемая структура для финального проекта. Можно адаптировать под свой стек, но важно сохранить идею: **точка входа, конфиг, тесты, CI/CD, Docker**.

## ⏱️ Сколько времени

Ориентир: **30–60 минут**, чтобы создать каркас и запустить “hello world”.

## 📁 Рекомендуемая структура

```
my-project/
├── README.md
├── requirements.txt
├── Dockerfile
├── docker-compose.yml
├── .github/
│   └── workflows/
│       └── deploy.yml
├── src/
│   ├── __init__.py
│   ├── main.py
│   └── config.py
└── tests/
    └── test_main.py
```

## ✅ Что важно (минимум)

- **`README.md`**: как запустить локально и как прогнать тесты
- **`src/main.py`**: точка входа приложения (или `app.py`, если так удобнее)
- **`tests/`**: хотя бы 2–3 базовых теста
- **`.github/workflows/`**: CI (tests) и CD (deploy)
- **`Dockerfile` + `docker-compose.yml`**: воспроизводимый запуск

## Артефакты

- Репозиторий с такой (или эквивалентной) структурой
- CI зелёный хотя бы на тестах
