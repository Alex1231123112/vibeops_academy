# Домашнее задание: Урок 4.4

## Задание: Собрать минимальный полный пайплайн

Соберите пайплайн по чек-листу из урока для своего или учебного проекта.

### Что должно быть

1. **CI: тесты на push и pull_request**  
   Workflow в `.github/workflows/`: на каждом push и PR запускаются тесты (например `pytest`). Шаги: checkout, setup Python, установка зависимостей, запуск тестов.

2. **CD: деплой на main**  
   Отдельный job или workflow: при push в `main` — деплой (Render, Railway или другой провайдер). Секреты (API key, deploy hook URL и т.п.) — только в GitHub Secrets, не в коде.

3. **Секреты**  
   В репозитории нет `.env`, токенов и паролей. Локально используете `.env` (в .gitignore). В CI/CD и на проде — переменные окружения из настроек платформы / GitHub Secrets.

4. **Health endpoint**  
   В приложении есть эндпоинт `/health`, возвращающий JSON (например `{"status": "healthy"}`). После деплоя проверьте его вручную или в workflow.

5. **Логи**  
   В приложении настроен базовый логгер (например `logging`). При старте и при ключевых действиях пишутся логи без секретов.

### Критерии сдачи

- [ ] CI workflow: тесты на push/PR
- [ ] CD workflow: деплой на main (или описание, если деплой вручную по шагам урока)
- [ ] Секреты только в GitHub Secrets / переменных окружения
- [ ] Есть `/health`, вы проверили его после деплоя
- [ ] Есть базовые логи, по которым можно понять, что происходит

Ссылка на репо с workflow-файлами и (по возможности) на работающее приложение по HTTPS.
