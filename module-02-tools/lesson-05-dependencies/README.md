## 📖 Теория (минимум)

### Почему нельзя “ставить всё в глобальный Python”

- проекты начинают конфликтовать по версиям пакетов;
- вы не можете воспроизвести окружение на другом компьютере/в CI;
- AI часто предлагает “поставь ещё 5 пакетов”, и без фиксации всё быстро превращается в хаос.

### Виртуальное окружение (venv)

Идея: у проекта есть свой набор библиотек.

Типовой набор команд:
- создать окружение: `python -m venv .venv`
- активировать (Windows): `.venv\\Scripts\\activate`
- активировать (macOS/Linux): `source .venv/bin/activate`

### requirements.txt

- фиксирует зависимости и версии (или хотя бы список пакетов);
- помогает CI и другим людям повторить запуск.
