# Шпаргалка: Промпт-инжиниринг (урок 2.4)

## Шаблон промпта (копируй-вставляй)

```
Контекст:
- проект/файл: ...
- стек/версии: ...

Цель:
- нужно: ...

Ограничения:
- нельзя: ...

Формат ответа:
- дай: (патч / шаги / код+тесты)

Критерии качества:
- edge cases: ...
- не ломай поведение: да/нет
```

## Супер-короткие усилители

- “Сначала задай до 7 уточняющих вопросов.”
- “Дай 3 edge cases и как их проверить.”
- “Покажи патч, а не пересказ.”
- “Не меняй логику, только читаемость.”
