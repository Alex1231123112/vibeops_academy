## 📖 Теория (практическая структура промпта)

Хороший промпт для программирования обычно содержит:

1) **Контекст**: что за проект/файл/ограничения  
2) **Цель**: что нужно получить в результате  
3) **Ограничения**: версии, зависимости, что нельзя делать  
4) **Формат ответа**: “дай патч”, “дай список шагов”, “дай код + тесты”  
5) **Критерии качества**: edge cases, читаемость, безопасность  

### Пример: плохой → хороший

Плохо:

```
Напиши мне Dockerfile
```

Хорошо:

```
Напиши Dockerfile для Python 3.12 приложения.
Требования:
- базовый образ: python:3.12-slim
- запуск: uvicorn app:app --host 0.0.0.0 --port 8000
- не запускай под root
- добавь healthcheck
Ответ: только содержимое Dockerfile.
```

### Итерации (как улучшать ответ)

- Если ответ “не туда” — уточните **контракт** (вход/выход), **ограничения** и **пример**.
- Просите AI добавить **краевые случаи** и **проверки**.
