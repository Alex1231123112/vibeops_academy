# Промпты для AI: Урок 2.4 (Промпт-инжиниринг)

## 1) Шаблон “контекст → цель → ограничения → формат”

```
Контекст:
- проект/файл: [...]
- версия/стек: [...]

Цель:
- нужно: [...]

Ограничения:
- нельзя: [...]
- должно быть совместимо с: [...]

Формат ответа:
- дай: (патч / список шагов / код + тесты)

Критерии качества:
- учти edge cases: [...]
- не ломай поведение: [...]
```

## 2) Отладка ошибки

```
Ошибка:
[вставь traceback/сообщение]

Код:
[вставь код]

1) Скажи, где именно причина (строка/функция) и почему.
2) Дай исправление.
3) Добавь 3 тест-кейса (или примера ввода), чтобы убедиться, что фикс работает.
```

## 3) Рефакторинг без смены логики

```
Вот код:
[код]

Сделай 2–3 улучшения читаемости:
- разбей на функции
- переименуй переменные
- добавь type hints
Не меняй поведение. Покажи патч.
```

## 4) “Сначала вопросы”

```
Я хочу решить задачу: [задача].
Сначала задай до 7 уточняющих вопросов (коротко), чтобы не ошибиться с требованиями.
Потом предложи план решения.
```
