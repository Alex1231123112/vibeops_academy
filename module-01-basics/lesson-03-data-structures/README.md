## 📖 Теория (минимум, который реально нужен)

### Когда что использовать

- **`list`**: порядок важен, бывают повторы. Пример: список задач.
- **`dict`**: быстрый доступ по ключу. Пример: настройки `{"host": "...", "port": 8000}`.
- **`set`**: только уникальные значения. Пример: уникальные категории товаров.

### Типичные операции

**Список:**

```python
tasks = ["a", "b", "c"]
tasks.append("d")
first = tasks[0]
for i, t in enumerate(tasks, start=1):
    print(i, t)
```

**Словарь:**

```python
prices = {"apple": 100, "banana": 80}

# Плохо (может KeyError):
# x = prices["orange"]

# Хорошо (без падения):
x = prices.get("orange")
if x is None:
    print("Нет такой позиции")
```

**Множество:**

```python
categories = {"tools", "books"}
categories.add("books")  # дубль не появится
```

### Типичные ошибки AI (и как их ловить)

- **KeyError / IndexError**: обращение к ключу/индексу без проверок → используйте `.get()`, проверяйте диапазон.
- **Мутация списка в цикле**: удаление элементов в `for` по тому же списку → лучше создавать новый список или идти по копии.
- **Путаница типов**: “числа в строке” после `input()` → приводите типы явно.
