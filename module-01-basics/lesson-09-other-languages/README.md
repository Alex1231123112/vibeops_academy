# Урок 1.9: Другие языки — JavaScript/TypeScript и Go

Курс построен на Python, но в реальных проектах и в ответах AI вы встретите и другие языки. Умение хотя бы **читать** код на JS/TS и Go и понимать, когда какой язык уместен, расширяет кругозор и помогает не пугаться чужого кода.

- Понять ключевые отличия от Python (типизация, ошибки, асинхронность, зависимости)
- Уметь попросить AI “перевести” небольшой фрагмент и проверить, что перевод корректный

## Зачем смотреть другие языки

- **Веб и фронтенд** — по умолчанию пишут на JavaScript/TypeScript; без них не понять типичный фронт или Node.js-бэкенд.
- **DevOps и инфраструктура** — многие инструменты (Kubernetes, Docker, Terraform, Prometheus) написаны на Go; скрипты и утилиты часто тоже на Go.
- **Код от AI** — нейросети часто предлагают примеры на JS/TS или Go; полезно уметь быстро ориентироваться в синтаксисе и отличиях от Python.

## Секция 1: JavaScript и TypeScript

### Минимум синтаксиса

- **Переменные:** `let` (можно менять), `const` (константа). Тип не указывается в JS; в TypeScript — опционально: `let x: number = 5;`.
- **Функции:** обычные `function add(a, b) { return a + b; }` и стрелочные `const add = (a, b) => a + b;`.
- **Типы в TypeScript:** `string`, `number`, `boolean`, массивы `string[]` или `Array<number>`, объекты через `interface` или `type`.
- **Условия и циклы:** `if/else`, `for`, `while` — как в Python, но с фигурными скобками `{}` и точкой с запятой в конце строк (часто опускают благодаря автоформатированию).

```javascript
// JavaScript
const items = ["a", "b", "c"];
items.forEach((item) => console.log(item));
```

```typescript
// TypeScript — типы для аргументов и возврата
function greet(name: string): string {
  return `Hello, ${name}`;
}
```

### Преимущества JS/TS

- **Веб и фронт** — язык браузера; React, Vue, Angular пишут на JS/TS.
- **Node.js** — один язык на фронте и бэкенде; огромная экосистема (npm).
- **Много примеров и кода от AI** — туториалы и ответы нейросетей часто на JS/TS.

### Отличия от Python

| Аспект | Python | JavaScript/TypeScript |
|--------|--------|------------------------|
| Синтаксис | Отступы, без `;` | Фигурные скобки `{}`, часто `;` |
| Типизация | Опциональные type hints | В JS — нет; в TS — статическая |
| Асинхронность | `async/await`, потоки | `async/await`, событийный цикл, промисы |
| Зависимости | `pip`, `requirements.txt` | `npm`, `package.json` |
| «Нет значения» | Один объект `None` | `undefined` и `null` |

## Секция 2: Go

### Минимум синтаксиса

- **Переменные:** `var x int = 5` или кратко `x := 5` (тип выводится). После объявления тип не меняется.
- **Функции:** возврат через `return`; несколько возвращаемых значений и ошибка — норма: `func ReadFile(name string) ([]byte, error)`.
- **Типы:** `string`, `int`, `int64`, `bool`, слайсы `[]string`, мапы `map[string]int`, структуры `struct { Name string }`.
- **Цикл:** только `for` (как `for i := 0; i < 10; i++` или `for _, item := range items`).
- **Ошибки:** не исключения, а значение. Почти всегда проверяют: `if err != nil { return err }`.

```go
package main

import "fmt"

func main() {
    items := []string{"a", "b", "c"}
    for _, s := range items {
        fmt.Println(s)
    }
}
```

```go
// Функция с ошибкой — типичный стиль Go
func LoadConfig(path string) (*Config, error) {
    data, err := os.ReadFile(path)
    if err != nil {
        return nil, err
    }
    // ...
}
```

### Преимущества Go

- **Один исполняемый файл** — после `go build` не нужны интерпретатор и зависимости в рантайме; удобно для деплоя и контейнеров.
- **Скорость** — компилируемый язык; быстрый старт и низкое потребление памяти по сравнению с интерпретируемым Python.
- **DevOps-стандарт** — Kubernetes, Docker, Terraform, Prometheus, многие CLI написаны на Go; знание Go помогает читать их код и писать свои утилиты.
- **Простой синтаксис** — мало ключевых слов, нет наследования в питоновском виде; быстрый вход для тех, кто уже знает основы программирования.

### Отличия от Python

| Аспект | Python | Go |
|--------|--------|-----|
| Выполнение | Интерпретатор | Компиляция в бинарник |
| Типизация | Динамическая, hints опциональны | Статическая, обязательная |
| ООП | Классы, наследование | Нет классов; есть структуры и методы у типов |
| Ошибки | Исключения (try/except) | Возврат `error` из функций, проверка `if err != nil` |
| Параллельность | Потоки (GIL), multiprocessing | Горутины (легковесные потоки), каналы |

## Когда что выбирать

- **Python** — скрипты, прототипы, ML/AI-пайплайны, автоматизация, быстрый старт; максимум готовых библиотек под аналитику и нейросети.
- **JavaScript/TypeScript** — веб-фронтенд, Node.js-бэкенд, fullstack на одном языке; когда проект уже в экосистеме npm.
- **Go** — CLI-утилиты, микросервисы, инструменты инфраструктуры (агенты, сборщики метрик), когда важны один бинарник и предсказуемая производительность.
