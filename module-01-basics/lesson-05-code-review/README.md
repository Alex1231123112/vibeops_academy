# Урок 1.5: Code Review

## 🎯 Цели урока
- Как проверять код от AI
- Чек-лист для самопроверки
- Типичные проблемы и как их находить

## ⏱️ Сколько времени

Ориентир: **45–90 минут** (пройтись по чек‑листу + написать короткий отчёт).

## ✅ Чек-лист code review (особенно для кода от AI)

Используйте этот список, когда AI сгенерировал код, вы взяли кусок из интернета или просто хотите убедиться, что всё ок.

### 1) Запуск и воспроизводимость

- [ ] Код запускается “с нуля” (после `git clone`) по понятной инструкции
- [ ] Нет “магии окружения” (пути, файлы, переменные), без которой всё ломается
- [ ] Если есть зависимости — они зафиксированы (например, `requirements.txt`)

### 2) Ввод, вывод и поведение

- [ ] Понятно, какие входные данные ожидаются (формат, диапазоны)
- [ ] Понятно, что программа должна вывести/вернуть
- [ ] Сообщения об ошибках понятные пользователю (а не “что-то пошло не так”)

### 3) Граничные случаи (то, где AI чаще всего ошибается)

- [ ] Пустой ввод / пустая строка
- [ ] Пустой список / словарь
- [ ] Нулевые значения, отрицательные числа
- [ ] Индексы вне диапазона, несуществующие ключи в словаре
- [ ] Деление на ноль

### 4) Логика и корректность

- [ ] Условия не “наоборот”, нет недостижимых веток
- [ ] Нет бесконечных циклов без выхода
- [ ] Нет скрытых зависимостей от глобальных переменных
- [ ] В сложных местах есть небольшие проверки/тестовые кейсы (хотя бы руками)

### 5) Структура и читаемость

- [ ] Имена переменных/функций отражают смысл (а не `a`, `data2`, `tmp`)
- [ ] Код разбит на функции (нет “простыни” на 200 строк)
- [ ] Нет явного дублирования — вынесено в функцию
- [ ] Типы/контракты понятны (type hints хотя бы на публичных функциях)

### 6) Безопасность и приватность (минимум)

- [ ] В коде нет токенов, паролей, ключей, `.env` и т.п.
- [ ] Если код читает/пишет файлы — пути валидируются, нет опасных удалений “вслепую”
- [ ] Если есть сеть/HTTP — таймауты и обработка ошибок присутствуют (хотя бы базово)

## Как сдавать ревью (формат)

Минимальный отчёт (3–5 пунктов):
- **что проверил**
- **какие баги/риски нашёл**
- **что исправил** (если исправлял)
