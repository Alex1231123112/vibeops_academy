# Промпты для безопасности (secrets, Docker, зависимости)

Перед использованием: см. общий шаблон промпта — [template.md](template.md).

## 1) Проверка утечек секретов

```
Проверь, нет ли риска утечки секретов в репозитории.
Вот список файлов/фрагментов:
[вставь]

Найди:
- токены, ключи, пароли, .env
- приватные URL (deploy hooks)
- подозрительные строки (JWT_SECRET, API_KEY и т.п.)

Ответ:
1) что опасно и почему
2) как исправить (gitignore, secrets, env vars)
3) как предотвратить повторение
```

## 2) Сделай Dockerfile безопаснее (минимум)

```
Вот Dockerfile:
[вставь]

Сделай безопаснее по минимуму:
- не root (создай пользователя и USER)
- добавь .dockerignore (список строк)
- предложи 2–3 улучшения (slim образ, меньше пакетов)

Ответ: патч + .dockerignore + тест-план (docker build/run).
```

## 3) Разбор отчёта Trivy

```
Вот вывод Trivy:
[вставь]

1) Сгруппируй HIGH и CRITICAL
2) Что можно исправить обновлением образа/пакетов
3) Что делать, если fixed version нет (unfixed)
4) Как зафиксировать риск (кратко)
```
