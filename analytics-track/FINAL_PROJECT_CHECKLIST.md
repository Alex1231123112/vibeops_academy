# Чек-лист финального проекта (Analytics Track)

## ⏱️ Сколько времени

Ориентир: **6–12 часов** на MVP.

## ✅ Чек-лист

### Репозиторий и воспроизводимость

- [ ] Код/анализ в GitHub
- [ ] README: цель, источник данных, как запустить, как воспроизвести, как обновлять
- [ ] `requirements.txt` / `pyproject.toml` (зависимости фиксированы)

### Данные и качество

- [ ] Явно описана схема данных (колонки/типы) или есть ссылка на источник
- [ ] Есть 5+ data checks (null/range/uniqueness/schema)

### Анализ и выводы

- [ ] Есть 3+ метрики/вывода, привязанные к вопросу
- [ ] Есть визуализации (минимум 3 графика)
- [ ] Есть рекомендации/следующие шаги (3–5 пунктов)

### Артефакт “для людей”

Выберите один:
- [ ] Markdown отчёт в `reports/`
- [ ] Dashboard (Streamlit/BI) + ссылка
- [ ] Notebook с итоговым summary (но лучше отдельно `report.md`)

### Автоматизация (по возможности)

- [ ] Минимальный CI: запуск checks/тестов/линта на push

## 🧪 Test plan

- [ ] В чистом окружении ставятся зависимости и запускается анализ по README
- [ ] Data checks проходят (или понятно, какие падают и почему)
- [ ] Артефакт обновляется (пересчёт отчёта/дашборда)
