# Домашнее задание: Урок 5.2 (Дизайн API)

## Задание: описать контракт мини-API

Выберите любой учебный сервис (или придумайте свой) и опишите API.

### Минимум (обязательно)

1) Список эндпоинтов (3–6 штук), например:
- `GET /health`
- `GET /items`
- `POST /items`
- `GET /items/{id}`

2) Для каждого эндпоинта:
- пример запроса
- пример ответа (status + JSON)
- минимум 1 ошибка (status + JSON)

### Формат сдачи

Вариант A: Markdown файл `API.md`  
Вариант B: OpenAPI YAML (черновик)

## Критерии сдачи

- [ ] Есть 3+ эндпоинта
- [ ] Есть примеры ответов и ошибок
- [ ] Есть 3 edge cases и как API должен отвечать
