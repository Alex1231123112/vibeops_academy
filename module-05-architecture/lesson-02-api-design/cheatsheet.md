# Шпаргалка: дизайн API (урок 5.2)

## Минимальный контракт API

- Список эндпоинтов: метод + путь
- Для каждого:
  - вход (query/path/body)
  - выход (status + JSON)
  - ошибки (400/404/500)

## Частые статусы

| Статус | Когда |
|---|---|
| 200 OK | успешный запрос |
| 201 Created | создали ресурс |
| 204 No Content | успешно, но без тела |
| 400 Bad Request | пользователь отправил плохие данные |
| 401 Unauthorized | нет авторизации |
| 403 Forbidden | авторизация есть, но прав нет |
| 404 Not Found | ресурс не найден |

## Edge cases (минимум)

- пустые поля / неправильные типы
- несуществующий id
- конфликт (например, уже существует)
- превышение лимитов (слишком длинная строка)
