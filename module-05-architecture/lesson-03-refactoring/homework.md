# Домашнее задание: Урок 5.3 (Рефакторинг)

## Задание: рефакторинг без изменения поведения

1) Возьмите свой учебный проект (или код на 100–300 строк).  
2) Выберите 2–3 улучшения, например:
- вынести повтор в функцию,
- разбить файл на модули,
- улучшить именование,
- добавить валидацию и понятные ошибки (без изменения логики).

3) Перед рефакторингом зафиксируйте “как было проверено”:
- тесты, или
- 3–5 ручных кейсов (ввод → ожидаемый результат).

4) Сделайте рефакторинг и снова выполните проверки.

## Критерии сдачи

- [ ] Есть список изменений (2–3 пункта)
- [ ] Есть проверка “до” и “после” (тесты или ручные кейсы)
- [ ] Есть коммит/PR с описанием
