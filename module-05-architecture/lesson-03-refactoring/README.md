## 📖 Теория (минимум)

### Рефакторинг ≠ “переписать всё”

Цель рефакторинга — улучшить код (читаемость, структуру, расширяемость) **без изменения поведения**.

### Безопасный рефакторинг

- маленькие шаги
- после каждого шага — проверка (тесты / ручные кейсы)
- фиксировать изменения коммитами
