# Шпаргалка: рефакторинг (урок 5.3)

## Правила безопасного рефакторинга

- Маленькие изменения
- Частые коммиты
- Проверка после каждого шага (тесты/ручные кейсы)

## Типовые “безопасные” улучшения

- вынести повторяющийся код в функцию
- переименовать переменные/функции
- разделить большой файл на модули
- добавить type hints на публичные функции

## Что просить у AI

- “покажи патч”
- “не меняй поведение”
- “перечисли edge cases и как проверить”
