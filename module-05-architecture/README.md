# Модуль 5: Архитектура

Проектирование приложений с AI — контекст, API, рефакторинг.

## Зачем этот модуль

- Научиться давать AI правильный **контекст проекта**, чтобы ответы были точнее и стабильнее
- Уметь описывать и проверять **контракт API** (эндпоинты, форматы, ошибки)
- Делать рефакторинг “без поломок”: сохранять поведение, добавлять проверки и тесты

## Артефакты по модулю (что должно получиться)

- README/контекст проекта (структура, запуск, ограничения, “что важно”)
- Черновик контракта API (список эндпоинтов или OpenAPI)
- PR/коммит с рефакторингом + описание “что изменил и как проверил, что не сломал”
