# Шпаргалка: Dockerfile (урок 3.2)

## Команды Dockerfile (минимум)

| Директива | Зачем |
|---|---|
| `FROM` | базовый образ |
| `WORKDIR` | рабочая директория внутри образа |
| `COPY` | копирование файлов |
| `RUN` | команды сборки (например, `pip install`) |
| `CMD` | команда запуска контейнера |

## Типовые команды Docker

| Что сделать | Команда |
|---|---|
| Собрать | `docker build -t my-app .` |
| Запустить | `docker run --rm my-app` |
| Запустить с портом | `docker run --rm -p 8000:8000 my-app` |
| Передать env | `docker run --rm -e DEBUG=1 my-app` |

## Частые проблемы

- `ModuleNotFoundError` внутри контейнера → зависимости не установлены или не туда скопировали.
- “не видит файл” → проверьте `WORKDIR` и пути `COPY`.
- “порт не доступен” → нужен `-p host:container` и приложение должно слушать `0.0.0.0`.
