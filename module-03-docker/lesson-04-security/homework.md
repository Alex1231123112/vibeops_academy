# Домашнее задание: Урок 3.4

## Задание: Применить чек-лист безопасности к своему образу

Возьмите Dockerfile из урока 3.2 (или своего проекта) и приведите его к чек-листу из этого урока.

### Что сделать

1. **USER (не root)**  
   Добавьте в Dockerfile создание пользователя и `USER appuser` (или другое имя). Убедитесь, что приложение запускается от этого пользователя (при необходимости скопируйте файлы до переключения или настройте права).

2. **.dockerignore**  
   Создайте `.dockerignore` и добавьте минимум: `.git`, `.env`, `__pycache__`, `*.pyc`, `.venv`, `venv`. Пересоберите образ и убедитесь, что сборка проходит.

3. **Базовый образ**  
   Используйте образ на основе `python:3.12-slim` (или актуальный slim). В `requirements.txt` по возможности зафиксируйте версии пакетов (`==`).

4. **Сканирование Trivy**  
   Соберите образ и запустите Trivy:
   ```bash
   trivy image --severity HIGH,CRITICAL --ignore-unfixed <ваш-образ>:latest
   ```
   Или через контейнер:
   ```bash
   docker run --rm -v /var/run/docker.sock:/var/run/docker.sock aquasec/trivy:latest image --severity HIGH,CRITICAL --ignore-unfixed <ваш-образ>:latest
   ```

### Критерии сдачи

- [ ] В Dockerfile есть `USER` (не root)
- [ ] Есть `.dockerignore` с перечисленными исключениями
- [ ] Используется slim-образ
- [ ] Trivy запущен; в отчёте или в комментарии указано: есть ли HIGH/CRITICAL, что сделали (обновили образ/пакеты или зафиксировали риск)

Приложите скрин вывода Trivy или фрагмент лога и ссылку на репо с обновлённым Dockerfile и .dockerignore.
