## 📖 Теория (минимум)

### Образ vs контейнер

- **Образ (image)** — “шаблон” (как ISO/снимок), из него создают контейнеры.
- **Контейнер (container)** — запущенный экземпляр образа (процесс(ы) + файловая система).

### Почему Docker решает боль “у меня работает”

- одинаковое окружение на любом компьютере/в CI;
- зависимости и версии фиксируются в образе;
- запуск “в одну команду”.

### Минимальный набор команд

- `docker build -t my-app .` — собрать образ
- `docker run --rm my-app` — запустить контейнер
- `docker ps` — список запущенных контейнеров
- `docker images` — список образов
